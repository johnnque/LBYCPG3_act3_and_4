OSI Model

The OSI (Open Systems Interconnection) Model acts as a framework that explains how different computer systems communicate and transfer data to each other. Developed by the International Organization for Standardization (ISO), it consists of seven layers, each with specific tasks and responsibilities.

The 7 Layers of the OSI Model

  • Application Layer (Layer 7) – Creates the data to be transferred.
  • Presentation Layer (Layer 6) – Formats, encrypts, and compresses data.
  • Session Layer (Layer 5) – Establishes, manages, and terminates connections.
  • Transport Layer (Layer 4) – Breaks data into segments for reliable delivery.
  • Network Layer (Layer 3) – Breaks segments into packets and handles routing.
  • Data Link Layer (Layer 2) – Converts packets into frames for transmission.
  • Physical Layer (Layer 1) – Transmits raw bits through physical media.

1. Encapsulation and Data Flow ▼

The OSI layers are numbered from bottom to top, with the Physical Layer as Layer 1 and the Application Layer as Layer 7.
When data is sent, it starts at the Application Layer and travels downward through the layers. At each layer, additional information (headers) is added — a process known as encapsulation. These headers help identify the destination and manage communication.
When the data reaches the receiving device, the process is reversed. The data moves upward through the layers, and each layer removes its corresponding header — a process called decapsulation.
Each layer acts as a checkpoint in the communication process. No layer can be skipped, and a failure in any layer may cause communication issues.

2. Protocols and Examples Per Layer ▼

The OSI Model uses different protocols at each layer to ensure proper communication between devices
The Application Layer uses protocols such as HTTP, FTP, SMTP and DNS which is used for web browsing, file transfer, and email services
The Presentation Layer handles the encryption and formatting through SSL/TLS and data formats like JPEG or MPEG
The Session Layer manages connections using protocols like NetBIOS and RPC.
The Transport Layer ensures delivery through TCP (reliable) and UDP (faster but connectionless).
The Network Layer routes data using IP and ICMP.
The Data Link Layer manages physical addressing through Ethernet and ARP
The Physical Layer transmit raw data using cables, fiber optics, or wireless signals
